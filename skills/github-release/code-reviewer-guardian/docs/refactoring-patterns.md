
# Refactoring Patterns (Fowler-inspired)

- **Extract Method / Extract Type**: break down long methods and god objects.
- **Introduce Parameter Object**: remove data clumps.
- **Replace Conditional with Polymorphism**: reduce branching and open for extension.
- **Move Method / Move Field**: fix feature envy.
- **Inline Variable / Decompose Conditional**: simplify control flow.
- **Encapsulate Collection**: avoid exposing mutable collections.
- **Separate Query from Modifier**: clarity and testability.

Always keep tests green; refactor in small steps.
