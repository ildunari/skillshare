
# PR Review Guide

**Author checklist**:
- Linked issue; clear description & screenshots.
- Tests updated; coverage stable or rising.
- Linting & formatting clean.
- Backward compatibility & migration steps described.

**Reviewer approach**:
- Start with high-level diff: scope and risk.
- Run analyzers; scan P0/P1 first.
- Check architecture boundaries and error handling.
- Leave specific, respectful feedback with code snippets.
