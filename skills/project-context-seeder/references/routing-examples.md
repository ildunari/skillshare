# Routing Examples

Use this file when you need extra routing clarity beyond the main routing table.

## Strong should-trigger examples
- "Make an AGENTS.md for this repo"
- "Review my CLAUDE.md and tell me if it's stale"
- "Set up this repo for Codex and Claude Code"
- "Prepare this codebase for AI agents"
- "Create a .cursorrules file using the actual repo structure"
- "Is my CLAUDE.md any good, or should I redo it from scratch?"
- "Update my CLAUDE.md to reflect all the changes we made this sprint"

## Strong should-not-trigger examples
- "Fix a typo in AGENTS.md"
- "Add one line to CLAUDE.md"
- "Update a single command in AGENTS.md"
- "Analyze this codebase for security vulnerabilities"
- "Write general documentation for this project"

## Boundary rule

Use the skill for whole-file create/rewrite/review/audit/refresh work.
Do not use it for tiny edits to an existing instruction file.
