# Skill Feedback: fixing-office-accessibility

Capture feedback from real runs here. Keep the list capped at 75 entries; when adding entry 76, remove or consolidate the oldest low-value item. Use category tags so repeated failure modes are easy to spot.

## Categories

- `[triggering]` Skill did or did not activate at the right time.
- `[triage]` Findings were grouped or ordered poorly.
- `[decision]` A framework led to a weak accessibility judgment.
- `[officecli]` A command was missing, wrong, or version-sensitive.
- `[interaction]` User questions were too many, unclear, or asked at the wrong time.
- `[safety]` Backup, validation, revert, or idempotency behavior failed.
- `[reporting]` Final human or JSON report was incomplete or hard to use.
- `[docs]` Reference material was missing or hard to navigate.

## Entry format

```text
- [category] YYYY-MM-DD — Summary of observed behavior. Expected behavior. Repro prompt or manifest path if available.
```
