# Creative Brief Workflow

Use this when the user's idea is loose, aesthetic, or gameplay-oriented.

## Intent Questions

Ask only what affects the asset:

1. What role does this sprite play in the game?
2. What camera/view will the player see it from?
3. What should the movement feel like?

If the user has already implied the answers, infer them and write assumptions in `brief.md`.

## Concept Output

For ideation, create 3-6 concepts using this structure:

```text
Name:
Silhouette:
Palette:
Motion promise:
Gameplay verbs:
Risk:
```

Keep concepts concrete. A usable concept says what would show up in the sprite sheet: scarf trail, lantern bob, flame squash, broken-hinge wobble, heavy recoil, hover pulse.

## Brief Template

```md
# <Asset Name>

## Goal
<What the asset does in the game.>

## Player Read
<What the player should understand in one glance.>

## Camera And Scale
<side-view/top-down/isometric/etc.; cell size; expected on-screen size>

## Art Direction
<style, palette, outline, material/rendering rules>

## Identity Locks
- <shape/marking/prop/detail that must persist>

## Animation Plan
<rows with frame counts and loop flags>

## Technical Notes
<background, alpha, engine, naming, timing>
```
