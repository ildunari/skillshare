# DOCX Enhanced — Feedback Log

> **MUST-READ when loading this skill.** This file captures observed issues, successful patterns, and improvement opportunities discovered during use. Read before building. Max 75 entries — compact old entries when approaching cap.

## Format

Each entry: `[CATEGORY] Date — observation (context)`

## Categories

- `[STYLE]` — Style system issues (missing styles, direct formatting leaks, inheritance problems)
- `[TYPOGRAPHY]` — Font pairing, sizing, spacing, or vertical rhythm issues
- `[TABLE]` — Table design, borders, padding, alignment, caption issues
- `[LAYOUT]` — Page geometry, margins, sections, headers/footers, orphans/widows
- `[DOMAIN]` — Domain constraint violations (NIH, manuscript, legal, etc.)
- `[TOOLCHAIN]` — Tool bugs, workarounds, version-specific issues (docx-js, python-docx, pandoc)
- `[OOXML]` — Raw XML editing issues (tracked changes, comments, numbering)
- `[FIGURE]` — Image sizing, caption placement, resolution, embedding issues
- `[LIST]` — List numbering, restart, multilevel, bullet style issues
- `[QA]` — Quality pipeline failures, missed checks, render discrepancies
- `[TEMPLATE]` — Template engineering, reference doc, style binding issues
- `[CONTENT]` — Content structure, heading depth, section balance, navigation decisions
- `[META]` — Skill structure, documentation, or process issues
