# Preset: crisp-human

Use when the user wants direct, no-fluff writing: technical notes, internal updates, product docs, incident summaries, instructions.

## Style

- Short to medium sentences. Occasional fragments are OK.
- Minimal adjectives. No hype.
- Prefer active voice and concrete verbs.
- Tight paragraphs (1–3 sentences each).
- Lists only when they genuinely help.

## Signature moves

- Start sentences with the verb when possible (“Cut…”, “Ship…”, “Fix…”).
- Delete any sentence that merely restates the previous one.
- Swap vague verbs for concrete ones (“improve” → “cut latency”, “reduce errors”, “ship faster”) when the source supports it.

## Do

- Lead with the point.
- Use plain words: “use,” “build,” “fix,” “ship,” “cut,” “raise,” “lower.”
- Remove throat-clearing and over-explanation.

## Don’t

- Don’t add personality or jokes unless the user’s text already has it.
- Don’t sound like marketing.
- Don’t over-polish into “corporate memo” voice.

## Anti-patterns to avoid

- Overusing transitions (“Moreover…”, “Additionally…”) instead of just moving to the next point.
- Paragraphs that all have the same length and shape.
- “In conclusion” recap endings — usually delete them.
