# Capability Taxonomy

Use this reference when comparing skills by what they actually do.

## Capability signals
- trigger phrases in description
- capability lines in SKILL.md
- bundled scripts
- bundled reference files
- supersedes declarations
- platform/runtime assumptions
- output artifact type

## Why this matters
Two skills can look similar by name but differ in:
- execution surface
- tooling assumptions
- output shape
- scripts/resources that make them unique

That should reduce false merge recommendations.
